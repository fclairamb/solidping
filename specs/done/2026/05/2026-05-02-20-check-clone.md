# Check clone — server-side `POST /checks/:uid/clone`

## Context

Today there is no built-in way to duplicate a check. A user who wants two near-identical checks (same SSH credentials, same SNMP `auth_password` + `priv_password`, same MQTT broker password, same `Authorization: Bearer …` header in HTTP, etc.) has to:

1. `GET /api/v1/orgs/:org/checks/:slug` from the dashboard.
2. Copy the returned config object (including secrets, which the API echoes back today — `server/internal/handlers/checks/service.go:441-616`).
3. Edit name/slug locally.
4. `POST /api/v1/orgs/:org/checks` with the modified body.

This works *only* because the dashboard currently sees plaintext credentials in the `GET` response. The spec at `specs/done/2026/05/2026-05-02-18-credentials-encryption.md` is going to remove that behaviour: post-encryption, `GET` will return a `configPrivateKeys: [...]` hint and **the actual secret values will never leave the server**. A frontend-driven clone built today would silently break the moment encryption lands — the user would clone a check and get a copy with empty `password`, `private_key`, `sasl_password`, etc. Worse, the failure mode is "check runs and reports `down`/`error`" rather than a clear validation error.

The same logic applies to per-check connection overrides (`check_connections.settings`) — the encryption spec scopes those for the same treatment (`IntegrationConnection.Settings` is in the in-scope list). So even setting credentials aside, the integration overrides are already a candidate for becoming server-only data.

Goal: a single endpoint that produces a near-identical copy of an existing check, server-side, including everything the user can't see (or shouldn't have to round-trip): config, regions, period, thresholds, adaptive settings, labels, check_group, **and check_connection rows with their per-check setting overrides**. No state — no results, no incidents, no scheduler row.

## Honest opinion

The user's instinct ("clone server-side because credentials") is right, and importantly it is the only design that survives the credentials-encryption work. Building it client-side now would create rework. Five concrete points beyond the headline:

**1. Default the clone to `enabled = false`.** This is the single most consequential decision and it should be the default, not an opt-in. Reasons:

- The user *just* cloned — odds are they want to tweak name, URL, or labels before letting it run. The natural workflow is: clone → land on the edit page → adjust → enable.
- Two enabled checks pointing at the same target (because the user hasn't changed the URL yet) doubles the load on the target and on Solidping's worker fleet for no benefit until the user finishes editing.
- If the source check is currently in an incident, the clone instantly inheriting `enabled=true` could fire a duplicate incident notification before the user touches anything.

Allow override via the request body (`{"enabled": true}`) for power users / API automation.

**2. Don't copy runtime state, ever.** Status, `status_streak`, `status_changed_at`, results, incidents, events, the `check_jobs` scheduler row — these are all "what has happened to *this* check". They reset to fresh on the clone, just like a freshly created check would. The scheduler row is created lazily by the scheduler loop — same as for a normal new check. No special handling needed; just don't carry these over.

**3. Don't auto-link the clone into status pages or maintenance windows.** A `status_page_resources` row points at a specific `check_uid`; same for `maintenance_windows.check_uid`. Tempting to copy these "for completeness", but:

- A status page is a deliberate publication scope. Silently exposing the clone (which probably has the wrong URL still) is a mini-incident.
- A maintenance window points at a specific check that the user knows is "under maintenance". Auto-attaching the clone could silently mute it.

These are explicit user actions; let the user re-link them. The clone metadata page should surface a "the source was on N status pages and M maintenance windows" hint so the user can act on it, but auto-linking is wrong.

**4. Copy check_connections, including their per-check `settings` overrides — but do *not* run the existing `auto-attach default connections` path.** `CreateCheck` (`service.go:594-601`) auto-attaches connections marked `default=true` for the org. For a clone, that's wrong: we want the *source's* connection set, not the org's defaults. The source may have removed a default connection deliberately, or added a non-default one with custom settings. Replicate exactly. This needs to be a different code path from `CreateCheck`.

**5. The endpoint shape: `POST /api/v1/orgs/:org/checks/:checkUid/clone`.** Two alternatives considered and rejected:

- **`POST /checks` with a `cloneFrom: "<uid>"` field.** Magic — readers of the API don't expect a creation endpoint to also do server-side credential copying. Hides the action.
- **`POST /checks/:uid/copies` (RESTful subcollection).** A "copy" isn't a sub-resource of the source; it's a sibling. No reason to model it as one.

`POST /:uid/clone` is an action verb, and the API already uses action verbs (`/checks/:uid/validate`, `/auth/switch-org`, `/auth/2fa/setup`). Consistent.

One thing **not** to confuse this with: the existing `ExportChecks` / `ImportChecks` endpoints (`server/internal/handlers/checks/handler.go:313-378`). Those are designed for cross-org / cross-instance migration via a JSON document the user downloads. Today they would echo plaintext secrets in the export, which is its own conversation (the encryption spec has them in scope: "no API surface that returns plaintext"). Clone is a different operation: a single in-org copy, secrets stay server-side throughout. Do not unify them.

## Scope

**In scope:**

- New endpoint `POST /api/v1/orgs/:org/checks/:checkUid/clone`. Auth: same middleware as the rest of `/checks` (`RequireAuth`).
- Optional JSON body with `name`, `slug`, `enabled`, `checkGroupUid`, `description` overrides. All fields optional; empty body is valid.
- New service method `Service.CloneCheck(ctx, orgSlug, sourceIdentifier, opts CloneCheckOptions) (CheckResponse, error)`.
- Server-side copy of: `name` (defaulted to `<source.name> (copy)`), `slug` (defaulted via `ensureUniqueSlug` with `-copy` suffix), `description`, `type`, `config` (verbatim — same JSONMap, same encrypted/plaintext shape), `regions`, `period`, `internal`, `incident_threshold`, `escalation_threshold`, `recovery_threshold`, `reopen_cooldown_multiplier`, `max_adaptive_increase`, `check_group_uid`, labels (re-link existing label rows to the new check_uid), check_connections (new rows with copied `settings` JSONMap).
- `enabled` defaults to **false**; overridable via body.
- Wrapped in a single DB transaction so partial clones don't leak rows on failure.
- Emits `check.created` event for the new check (so connections fire normally if the user enables it).
- CLI: `./solidping client checks clone <slug-or-uid> [--name=…] [--slug=…] [--enabled]`.
- MCP tool: `clone_check` (mirrors API).
- Tests: handler tests (auth, body parsing, error mapping), service tests (every cloneable field, label re-link, connection re-link with settings, runtime-state non-copy, slug conflict path, source-not-found, custom name/slug overrides, enabled default).

**Out of scope (deliberate):**

- Cloning across organizations (use export/import for that).
- Bulk clone (`?count=10`). Easy follow-up if needed; not asked for.
- Auto-renaming pattern beyond `(copy)` suffix (e.g., a numbered suffix like `(copy 2)`). The slug uniqueness loop will handle slug collisions; the user can rename.
- Status-page / maintenance-window auto-linking (see Honest Opinion #3).
- "Suggest related labels to add to the clone" or any other smartness.
- Any frontend dash0 UI work beyond a single "Clone" action button on the check detail/edit page that calls the new endpoint and navigates to the edit page of the new check. (The frontend wiring is small enough to live in this spec; I list it under Files touched.)
- Cloning into a *different* org. Cross-org clone is a re-encryption / re-DEK problem (see encryption spec) and should be its own follow-up.

## Approach

### 1. Endpoint

Register in `server/internal/app/server.go` next to the other `/checks/:checkUid` routes (around line 380):

```go
orgChecks.POST("/:checkUid/clone", checksHandler.CloneCheck)
```

`checkUid` is the existing path param convention (UID *or* slug — `GetCheckByUidOrSlug` resolves either; `handler.go:197` already does this).

### 2. Handler

Add to `server/internal/handlers/checks/handler.go`:

```go
// CloneCheck handles POST /api/v1/orgs/:org/checks/:checkUid/clone.
func (h *Handler) CloneCheck(writer http.ResponseWriter, req bunrouter.Request) error {
    orgSlug := req.Param("org")
    identifier := req.Param("checkUid")

    var cloneReq CloneCheckRequest
    // Empty body is valid — decode but ignore EOF.
    if err := json.NewDecoder(req.Body).Decode(&cloneReq); err != nil && !errors.Is(err, io.EOF) {
        return h.WriteValidationError(writer, "Invalid JSON", []base.ValidationErrorField{
            {Name: fieldBody, Message: msgInvalidJSON},
        })
    }

    check, err := h.svc.CloneCheck(req.Context(), orgSlug, identifier, &cloneReq)
    if err != nil {
        return h.handleCloneError(writer, err)
    }

    return h.WriteJSON(writer, http.StatusCreated, check)
}
```

`handleCloneError` is a thin switch over: `ErrOrganizationNotFound`, `ErrCheckNotFound`, `ErrSlugConflict` (only fires when the user provided an explicit `slug`), `ErrInvalidSlugFormat`. Internal errors fall through to `WriteInternalError`. Reuses existing error codes — no new ones.

### 3. Request type

Add to `server/internal/handlers/checks/service.go` near the other request types:

```go
// CloneCheckRequest represents a request to clone an existing check.
// All fields are optional — an empty body produces a clone with sensible defaults.
type CloneCheckRequest struct {
    Name          *string `json:"name,omitempty"`
    Slug          *string `json:"slug,omitempty"`
    Description   *string `json:"description,omitempty"`
    CheckGroupUID *string `json:"checkGroupUid,omitempty"`
    Enabled       *bool   `json:"enabled,omitempty"` // defaults to false
}
```

Using pointer types so we can distinguish "absent" from "empty string" (which `ensureUniqueSlug` and the name-default logic both care about).

### 4. Service method

```go
// CloneCheck creates a near-identical copy of an existing check, server-side.
// Runtime state (status, results, incidents, scheduler) is not copied.
// Status-page and maintenance-window references are not copied.
//
//nolint:cyclop,funlen // Lots of fields to copy; flat is clearer than helpers here.
func (s *Service) CloneCheck(
    ctx context.Context, orgSlug, sourceIdentifier string, req *CloneCheckRequest,
) (CheckResponse, error) {
    org, err := s.db.GetOrganizationBySlug(ctx, orgSlug)
    if err != nil {
        return CheckResponse{}, ErrOrganizationNotFound
    }

    source, err := s.db.GetCheckByUidOrSlug(ctx, org.UID, sourceIdentifier)
    if err != nil || source == nil {
        return CheckResponse{}, ErrCheckNotFound
    }

    // Build the new check.
    sourceSlug := ""
    if source.Slug != nil {
        sourceSlug = *source.Slug
    }

    // Resolve target slug.
    var (
        targetSlug      string
        userProvidedSlug bool
    )
    switch {
    case req != nil && req.Slug != nil && *req.Slug != "":
        if err := validateSlug(*req.Slug); err != nil {
            return CheckResponse{}, err
        }
        targetSlug = *req.Slug
        userProvidedSlug = true
    default:
        // Auto-generate from source slug.
        base := sourceSlug
        if base == "" {
            base = source.Type
        }
        targetSlug = base + "-copy"
    }

    finalSlug, err := s.ensureUniqueSlug(ctx, org.UID, targetSlug, userProvidedSlug)
    if err != nil {
        return CheckResponse{}, err
    }

    // Resolve target name.
    sourceName := ""
    if source.Name != nil {
        sourceName = *source.Name
    }
    var targetName string
    switch {
    case req != nil && req.Name != nil && *req.Name != "":
        targetName = *req.Name
    case sourceName != "":
        targetName = sourceName + " (copy)"
    default:
        targetName = finalSlug
    }

    clone := models.NewCheck(org.UID, finalSlug, source.Type)
    clone.Name = &targetName

    // Copy all "configuration" fields. Runtime state stays at NewCheck defaults.
    if source.Description != nil {
        copied := *source.Description
        clone.Description = &copied
    }
    if req != nil && req.Description != nil {
        copied := *req.Description
        clone.Description = &copied
    }
    clone.Config = source.Config // JSONMap is a map — pass-by-reference is fine since we never mutate it
    clone.Regions = append([]string(nil), source.Regions...)
    clone.Period = source.Period
    clone.Internal = source.Internal
    clone.IncidentThreshold = source.IncidentThreshold
    clone.EscalationThreshold = source.EscalationThreshold
    clone.RecoveryThreshold = source.RecoveryThreshold
    clone.ReopenCooldownMultiplier = source.ReopenCooldownMultiplier
    clone.MaxAdaptiveIncrease = source.MaxAdaptiveIncrease

    // Check group: prefer override; else inherit source.
    switch {
    case req != nil && req.CheckGroupUID != nil:
        if *req.CheckGroupUID == "" {
            clone.CheckGroupUID = nil // explicit "remove from group"
        } else {
            v := *req.CheckGroupUID
            clone.CheckGroupUID = &v
        }
    default:
        clone.CheckGroupUID = source.CheckGroupUID
    }

    // Enabled: default false, override via request.
    clone.Enabled = false
    if req != nil && req.Enabled != nil {
        clone.Enabled = *req.Enabled
    }

    // Persist clone + labels + check_connections in a single transaction.
    err = s.db.RunInTx(ctx, func(txCtx context.Context) error {
        if err := s.db.CreateCheck(txCtx, clone); err != nil {
            return err
        }

        // Re-link existing labels to the new check.
        labels, err := s.db.GetLabelsForCheck(txCtx, source.UID)
        if err != nil {
            return err
        }
        if len(labels) > 0 {
            labelUIDs := make([]string, 0, len(labels))
            for _, lbl := range labels {
                labelUIDs = append(labelUIDs, lbl.UID)
            }
            if err := s.db.SetCheckLabels(txCtx, clone.UID, labelUIDs); err != nil {
                return err
            }
        }

        // Copy check_connections (NOT the auto-default-attach path).
        srcConns, err := s.db.ListCheckConnections(txCtx, source.UID)
        if err != nil {
            return err
        }
        for _, sc := range srcConns {
            cc := models.NewCheckConnection(clone.UID, sc.ConnectionUID, org.UID)
            if sc.Settings != nil {
                copied := make(models.JSONMap, len(*sc.Settings))
                for k, v := range *sc.Settings {
                    copied[k] = v
                }
                cc.Settings = &copied
            }
            if err := s.db.CreateCheckConnection(txCtx, cc); err != nil {
                return err
            }
        }

        return nil
    })
    if err != nil {
        return CheckResponse{}, err
    }

    // Emit check.created event (best-effort, like CreateCheck).
    if err := s.emitEvent(ctx, org.UID, models.EventTypeCheckCreated, clone); err != nil {
        slog.WarnContext(ctx, "failed to emit check.created event for clone", "error", err)
    }

    response := s.convertCheckToResponse(clone)
    // Attach labels to the response.
    if labels, err := s.db.GetLabelsForCheck(ctx, clone.UID); err == nil && len(labels) > 0 {
        response.Labels = make(map[string]string, len(labels))
        for _, lbl := range labels {
            response.Labels[lbl.Key] = lbl.Value
        }
    }
    return response, nil
}
```

Notes on the implementation:

- `RunInTx` — verify this exists on the db service; if not, the transaction wrapper is the trivial `db.RunInTx(ctx, func(ctx context.Context) error { … })` pattern over `bun.Tx`. If absent, add it as part of this work (one-line pass-through).
- `ListCheckConnections(checkUID)` — verify this method exists on the db layer; if it lives only in the `checkconnections` service today, expose it on the db layer (it's a single-table query).
- The `Config` JSONMap is **copied by reference**, not deep-cloned. This is safe because (a) we never mutate `source.Config` in `CloneCheck`, (b) Bun re-marshals the JSONMap when persisting the new check, so the two rows are independent on disk. If a future change starts mutating `clone.Config` in place, switch to a `maps.Clone` (Go 1.21+).
- **Post-encryption (specs/done/2026/05/2026-05-02-18-credentials-encryption.md):** when that spec lands, `Check` will gain a `ConfigPrivate` column. The clone code needs one extra line in the field-copy section: `clone.ConfigPrivate = source.ConfigPrivate`. The encrypted blob is keyed by the org DEK, and both source and clone are in the same org, so the same DEK decrypts both. **No re-encryption needed.** This is the single biggest reason the server-side approach is worth it: the clone code stays one line of additional copy. A frontend-driven clone would have to be redesigned end-to-end.

### 5. Error handling

```go
func (h *Handler) handleCloneError(writer http.ResponseWriter, err error) error {
    switch {
    case errors.Is(err, ErrOrganizationNotFound):
        return h.WriteErrorErr(writer, http.StatusNotFound,
            base.ErrorCodeOrganizationNotFound, "Organization not found", err)
    case errors.Is(err, ErrCheckNotFound):
        return h.WriteErrorErr(writer, http.StatusNotFound,
            base.ErrorCodeCheckNotFound, "Source check not found", err)
    case errors.Is(err, ErrSlugConflict):
        return h.WriteValidationError(writer, "Slug already exists",
            []base.ValidationErrorField{{Name: fieldSlug,
                Message: "A check with this slug already exists in this organization"}})
    case errors.Is(err, ErrInvalidSlugFormat):
        return h.WriteValidationError(writer, "Invalid slug format",
            []base.ValidationErrorField{{Name: fieldSlug, Message: invalidSlugMessage}})
    default:
        return h.WriteInternalError(writer, err)
    }
}
```

(Hoist the slug-format message to a constant if it isn't one already — three handlers now repeat it.)

### 8. Frontend (dash0)

Minimal: a "Clone" item in the check actions menu on `routes/orgs/$org/checks.$checkUid.tsx` (and optionally on the list row's overflow menu). Calls a new `useCloneCheck` mutation hook in `web/dash0/src/api/hooks.ts` that POSTs to `/api/v1/orgs/$org/checks/$checkUid/clone` with optional name, then navigates to `/dash0/orgs/$org/checks/$newSlug/edit`.

i18n: one key `checks.actions.clone` in `web/dash0/src/locales/{en,fr,de,es}/checks.json`.

UX detail: don't open a modal asking for a name. Just clone with the default `(copy)` suffix, navigate to edit, and the user can rename in the form they're already editing. One click, one redirect. If the user wants to name the clone before it exists, they can use the API/CLI.
