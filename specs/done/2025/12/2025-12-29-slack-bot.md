# Slack Integration

## Overview

Implement a Slack integration that enables:
1. **OAuth flow** to connect Slack workspaces to SolidPing organizations
2. **Inbound features** - slash commands, shortcuts, events, and link unfurling
3. **Connection management** via the `integration_connections` table

> **Note:** Incident notifications (sending alerts when checks fail/recover) are implemented separately in the **Notification System** (`specs/2026-01-01-notifiers.md`). This spec focuses on the Slack-specific infrastructure; the notification system uses the Slack connection created here to send messages.

## Features

### Incident Notifications

Handled by the Notification System (`specs/2026-01-01-notifiers.md`):
- Slack connections created via OAuth are stored in `integration_connections`
- The notification system creates jobs with `type = "notification"` when incidents occur
- A Slack sender uses the stored `access_token` to post messages

This spec does **not** implement the notification workflow itself.

### Slash Commands
- `/check <url>` - Create a new check for the given URL

### Shortcuts
- "Add an endpoint check" - Global shortcut to create a check via modal

### Events
| Event | Description |
|-------|-------------|
| `app_home_opened` | Display dashboard in App Home tab |
| `app_mention` | Respond when @mentioned |
| `app_uninstalled` | Cleanup when workspace removes the app |
| `link_shared` | Unfurl SolidPing links with rich previews |

### Link Unfurling
Unfurl links from `solidping.io` and `solidping.k8xp.com` with check status previews.

## API Structure

Two types of endpoints with different naming conventions:

### Inbound Endpoints (`/integrations`)

Called by Slack. No org prefix since Slack doesn't know our org structure.
The org is resolved internally via stored OAuth tokens.

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/integrations/slack/oauth` | OAuth callback |
| POST | `/api/v1/integrations/slack/events` | Event subscriptions |
| POST | `/api/v1/integrations/slack/command` | Slash commands |
| POST | `/api/v1/integrations/slack/interaction` | Interactivity (buttons, modals, shortcuts) |

### Management Endpoints (`/connections`)

Called by users to manage their org's connections. Each connection has a UID, allowing multiple connections per integration type (e.g., multiple Slack workspaces).

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v1/orgs/$org/connections` | List all connections (filter with `?type=slack`) |
| POST | `/api/v1/orgs/$org/connections` | Create a new connection (type in body) |
| GET | `/api/v1/orgs/$org/connections/$uid` | Get connection details |
| PATCH | `/api/v1/orgs/$org/connections/$uid` | Update settings (default channel, etc.) |
| DELETE | `/api/v1/orgs/$org/connections/$uid` | Delete connection |

## OAuth Scopes

### Bot Scopes
| Scope | Purpose |
|-------|---------|
| `chat:write` | Send messages to channels the bot is in |
| `chat:write.public` | Send messages to any public channel |
| `channels:read` | List public channels for channel picker |
| `groups:read` | List private channels the bot is in |
| `users:read` | Get user info for mentions |
| `users:read.email` | Match Slack users to SolidPing users |
| `team:read` | Get workspace info |
| `commands` | Handle slash commands |
| `app_mentions:read` | Receive @mention events |
| `reactions:write` | Add reactions to messages |
| `links:read` | Receive link_shared events for unfurling |

### User Scopes (for Sign in with Slack)
- `openid`, `profile`, `email`

## Database Structure

### integration_connections table

Generic table for all integration connections (Slack, BetterStack, webhooks, etc.).

| Column | Type | Description |
|--------|------|-------------|
| `uid` | UUID | Primary key |
| `organization_uid` | UUID | Foreign key to organizations |
| `type` | VARCHAR(50) | Integration type: `slack`, `betterstack`, `webhook`, etc. |
| `name` | VARCHAR(255) | User-defined name for the connection |
| `enabled` | BOOLEAN | Whether the connection is active |
| `is_default` | BOOLEAN | If true, auto-attach to new checks for notifications |
| `settings` | JSONB | Type-specific configuration (see below) |
| `created_at` | TIMESTAMP | Creation timestamp |
| `updated_at` | TIMESTAMP | Last update timestamp |
| `deleted_at` | TIMESTAMP | Soft delete |

**Indexes:**
- `idx_integration_connections_org_type` on `(organization_uid, type)`

### Slack-specific settings (JSON)

Stored in the `settings` JSONB column when `type = 'slack'`:

```json
{
  "team_id": "T0123456789",
  "team_name": "Acme Corp",
  "bot_user_id": "U0123456789",
  "access_token": "xoxb-...",
  "default_channel_id": "C0123456789",
  "default_channel_name": "#alerts",
  "installed_by_user_id": "U9876543210",
  "scopes": ["chat:write", "channels:read", "..."]
}
```

### integration_user_mappings table

Links external users to SolidPing users for attribution and mentions.

| Column | Type | Description |
|--------|------|-------------|
| `uid` | UUID | Primary key |
| `organization_uid` | UUID | Foreign key to organizations |
| `connection_uid` | UUID | Foreign key to integration_connections |
| `connection_user_id` | TEXT | User ID on the remote system (e.g., `U0123456789` for Slack) |
| `properties` | JSONB | User properties from the remote system (email, name, avatar, etc.) |
| `user_uid` | UUID | Foreign key to users (nullable, set when matched to local user) |
| `created_at` | TIMESTAMP | Creation timestamp |
| `updated_at` | TIMESTAMP | Last update timestamp |

**Indexes:**
- `idx_integration_user_mappings_connection_user` on `(connection_uid, connection_user_id)` UNIQUE
- `idx_integration_user_mappings_org` on `(organization_uid)`
- `idx_integration_user_mappings_user` on `(user_uid)`

### Lookup Patterns

**Find connection by Slack team_id** (for inbound webhooks):
```sql
SELECT * FROM integration_connections
WHERE type = 'slack'
AND settings->>'team_id' = $1;
```

**List all Slack connections for an org**:
```sql
SELECT * FROM integration_connections
WHERE organization_uid = $1 AND type = 'slack';
```
