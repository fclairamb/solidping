# Goal
Update the existing `CheckNotifier` interface to make it more flexible and support multiple event types with payloads.

## Motivation
The current `CheckNotifier` interface is limited to check creation events only. A more generic event notification system will allow:
- Supporting multiple event types (check created, check updated, check deleted, etc.)
- Passing event-specific data through payloads
- Easier extension for future event types
- Better separation of concerns between event producers and consumers

## Interface Changes

### Current Interface
```go
type CheckNotifier interface {
	// NotifyCheckCreated signals that a new check was created.
	// This is a best-effort notification - errors are ignored.
	NotifyCheckCreated(ctx context.Context) error

	// WaitChannel returns a channel that receives notifications.
	// Checkrunners can select on this channel to wake up early.
	WaitChannel() <-chan struct{}

	// Close releases resources used by the notifier.
	Close() error
}
```

### New Interface
```go
type EventNotifier interface {
	// Notify sends an event of the specified type with a payload.
	// This is a best-effort notification - errors may be logged but not propagated.
	Notify(ctx context.Context, eventType string, payload string) error

	// Listen returns a channel that receives payloads for events of the specified type.
	// Consumers can select on this channel to wake up when events occur.
	Listen(eventType string) <-chan string

	// Close releases resources used by the notifier.
	Close() error
}
```

**Note**: Changed parameter name from `type` to `eventType` since `type` is a reserved keyword in Go.

## Implementation Tasks
1. **Update interface definition** in the appropriate package
2. **Update PostgreSQL implementation** to:
   - Support generic event types instead of just check creation
   - Store and route payloads correctly
   - Handle multiple event type subscriptions
3. **Update in-memory/channel-based implementation** to:
   - Support multiple event types with separate channels or filtering
   - Pass payloads through channels
4. **Migrate existing usages** of `CheckNotifier` to use `EventNotifier`
5. **Update tests** to cover new interface functionality

## Example Usage
```go
// Producer side
notifier.Notify(ctx, "check.created", `{"check_uid": "chk_123"}`)
notifier.Notify(ctx, "check.updated", `{"check_uid": "chk_123", "status": "up"}`)

// Consumer side
checkCreatedChan := notifier.Listen("check.created")
checkUpdatedChan := notifier.Listen("check.updated")

select {
case payload := <-checkCreatedChan:
	// Handle check created event
case payload := <-checkUpdatedChan:
	// Handle check updated event
}
```
