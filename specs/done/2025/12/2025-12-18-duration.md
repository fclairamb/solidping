# Change Check.Period from String to time.Duration

## Overview
Migrate the `Period` field in the `Check` and `CheckJob` models from a string type to Go's native `time.Duration` type to improve type safety, parsing reliability, and code maintainability.

## Current State

### Database Layer
- **PostgreSQL**: `period interval not null default '1 minute'` ✅ Already correct type!
- **SQLite**: `period text not null default 'PT1M'` ❌ Needs migration

### Go Models
- **Check.Period**: `string` with bun tag `period,notnull,default:'PT1M'` (models.go:289)
- **CheckJob.Period**: `string` with bun tag `period,notnull` (models.go:331)

### API Layer
- **OpenAPI**: `type: string` with description "Interval duration (e.g., "00:01:00" for 1 minute)"
- **File**: `back/internal/app/openapi/openapi.yaml:1269-1271`

### Current Usage
- Code sets: `check.Period = "PT1M"` (job_startup.go:230)
- Parsing: `parseISO8601Duration(checkJob.Period)` (runner.go:600)

## Motivation
Using `string` for duration values has several drawbacks:
1. **No type safety** - Invalid duration strings can be stored without compile-time validation
2. **Runtime parsing errors** - Duration parsing happens at runtime, potentially causing failures
3. **Complex conversions** - Requires manual parsing/formatting between ISO 8601 and Go durations
4. **Less intuitive** - ISO 8601 format (`PT1M`) is less readable than Go's duration format (`time.Minute`)
5. **Error-prone** - Easy to introduce bugs when manipulating duration values
6. **Unnecessary code** - Custom `parseISO8601Duration()` function can be removed

### Step 8: Handle JSON Marshaling

Bun ORM should automatically handle the conversion between:
- PostgreSQL `interval` ↔ Go `time.Duration`
- SQLite `text` ↔ Go `time.Duration` (as string like "1m")

For JSON API responses, Go's `time.Duration` marshals to nanoseconds by default. To use readable format, add custom marshaling:

**File**: `back/internal/db/models/models.go`

Add after the Check struct definition:
```go
// MarshalJSON customizes JSON marshaling for Check to output duration in readable format
func (c *Check) MarshalJSON() ([]byte, error) {
	type Alias Check
	return json.Marshal(&struct {
		Period string `json:"period"`
		*Alias
	}{
		Period: c.Period.String(),
		Alias:  (*Alias)(c),
	})
}

// UnmarshalJSON customizes JSON unmarshaling to parse duration strings
func (c *Check) UnmarshalJSON(data []byte) error {
	type Alias Check
	aux := &struct {
		Period string `json:"period"`
		*Alias
	}{
		Alias: (*Alias)(c),
	}
	if err := json.Unmarshal(data, &aux); err != nil {
		return err
	}
	if aux.Period != "" {
		duration, err := time.ParseDuration(aux.Period)
		if err != nil {
			return fmt.Errorf("invalid period format: %w", err)
		}
		c.Period = duration
	}
	return nil
}
```

Apply the same pattern to `CheckJob` if needed.
