# DNS Check Specification

## Overview
The DNS checker performs DNS resolution queries to verify that hostnames resolve correctly. This is useful for monitoring DNS server health, validating DNS records, and detecting DNS configuration issues.

## Implementation
- Based on Go's standard `net` package (Resolver)
- Checker type: `"dns"`
- Package location: `/back/internal/checkers/checkdns/`

## Configuration

### Required Fields
- `hostname` (string) - The hostname to resolve (e.g., "example.com", "www.google.com")

### Optional Fields
- `timeout` (duration, default: 5s) - Maximum time to wait for DNS resolution
- `nameserver` (string, optional) - Custom DNS server to query (e.g., "8.8.8.8:53", "1.1.1.1:53"). If not specified, uses system default DNS
- `record_type` (string, default: "A") - DNS record type to query: "A", "AAAA", "CNAME", "MX", "NS", "TXT", "SOA"
- `expected_ips` ([]string, optional) - List of expected IP addresses. If specified, check fails if resolved IPs don't match
- `expected_values` ([]string, optional) - List of expected values for non-IP records (CNAME, MX, NS, TXT, SOA)

## Validation Rules

### Config Validation (Validate method)
- `hostname` must not be empty
- `timeout` must be > 0 and <= 60s
- `record_type` must be one of: "A", "AAAA", "CNAME", "MX", "NS", "TXT", "SOA"
- `nameserver` if specified, must be in format "host:port" (e.g., "8.8.8.8:53")
- Cannot specify both `expected_ips` and `expected_values` (use one or the other based on record type)

## Execution Behavior

### Success Criteria (StatusUp)
- DNS query completes successfully within timeout
- At least one record returned
- If `expected_ips` specified: resolved IPs match the expected list (order doesn't matter)
- If `expected_values` specified: resolved values match the expected list

### Failure Criteria (StatusDown)
- DNS query completes but returns no records (NXDOMAIN)
- Resolved IPs/values don't match expected values (if specified)

### Timeout (StatusTimeout)
- Context deadline exceeded before DNS response received

### Error (StatusError)
- Invalid nameserver format
- DNS server unreachable
- Invalid DNS response format
- Network errors

## Metrics

Returned in `Result.Metrics` (for time-series aggregation):
- `query_time_ms` (float64) - DNS query resolution time in milliseconds
- `record_count` (int) - Number of records returned
- `ttl_seconds` (int) - TTL of the DNS record (if available)

## Output

Returned in `Result.Output` (for diagnostic information):
- `hostname` (string) - The hostname that was queried
- `record_type` (string) - The DNS record type queried
- `resolved_ips` ([]string) - List of resolved IP addresses (for A/AAAA records)
- `resolved_values` ([]string) - List of resolved values (for CNAME, MX, NS, TXT, SOA records)
- `nameserver` (string) - The DNS server used (if custom nameserver specified)
- `error` (string) - Error message if check failed

## Implementation Notes

### DNS Resolution
- Use `net.Resolver` with custom `Dial` function if nameserver specified
- For default system DNS, use `&net.Resolver{}` without custom dialer
- Support context cancellation for timeout handling

### Record Types
- **A**: IPv4 address lookup (returns IP addresses)
- **AAAA**: IPv6 address lookup (returns IP addresses)
- **CNAME**: Canonical name (returns domain names)
- **MX**: Mail exchange (returns mail server hostnames with priority)
- **NS**: Name server (returns authoritative name servers)
- **TXT**: Text records (returns text strings)
- **SOA**: Start of authority (returns SOA record details)

### Expected Values Matching
- For IP records (A/AAAA): Compare resolved IPs with `expected_ips` (case-insensitive, order doesn't matter)
- For other records: Compare resolved values with `expected_values` (exact match, order doesn't matter)
- If expected values not specified, any non-empty result is considered success

### Error Handling
- Handle NXDOMAIN (non-existent domain) as StatusDown, not StatusError
- Handle timeout as StatusTimeout
- Handle network errors, invalid nameserver, etc. as StatusError
- Provide clear error messages for troubleshooting

## Examples

### Basic A Record Check
```json
{
  "hostname": "example.com",
  "record_type": "A"
}
```

### Check with Custom DNS Server
```json
{
  "hostname": "google.com",
  "nameserver": "8.8.8.8:53",
  "record_type": "A",
  "timeout": "3s"
}
```

### Check with Expected IPs
```json
{
  "hostname": "example.com",
  "record_type": "A",
  "expected_ips": ["93.184.216.34"]
}
```

### MX Record Check
```json
{
  "hostname": "gmail.com",
  "record_type": "MX"
}
```

### CNAME Check
```json
{
  "hostname": "www.example.com",
  "record_type": "CNAME",
  "expected_values": ["example.com"]
}
```
