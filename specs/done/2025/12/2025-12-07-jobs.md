# Job System Specification

## Overview

This document describes the implementation of a distributed job processing system for SolidPing, inspired by the RealAssets job system architecture. The system will provide asynchronous task execution with retry support, logging, and multi-tenancy.

## Goals

1. **Separation of concerns**: Clear boundaries between worker runner, database operations, job interface, and job implementations
2. **Extensibility**: Easy addition of new job types via registry pattern
3. **Reliability**: Retry support, atomic job claiming, and crash recovery
4. **Observability**: Structured logging with job context
5. **Multi-tenancy**: All jobs scoped to organizations

// JobDefinition is a factory for creating job instances.
// Each job type must implement this interface and register itself.

    Type() JobType

### Registry (`registry.go`)

```go
package jobdef

import (
    "fmt"
    "sync"
)

// Registry holds all registered job definitions.
type Registry struct {
    mu          sync.RWMutex
    definitions map[JobType]JobDefinition
}

// NewRegistry creates a new empty registry.
func NewRegistry() *Registry {
    return &Registry{
        definitions: make(map[JobType]JobDefinition),
    }
}

// Register adds a job definition to the registry.
// Panics if the job type is already registered.
func (r *Registry) Register(def JobDefinition) {
    r.mu.Lock()
    defer r.mu.Unlock()

    if _, exists := r.definitions[def.Type()]; exists {
        panic(fmt.Sprintf("job type %q already registered", def.Type()))
    }
    r.definitions[def.Type()] = def
}

// Get retrieves a job definition by type.
func (r *Registry) Get(jobType JobType) (JobDefinition, bool) {
    r.mu.RLock()
    defer r.mu.RUnlock()

    def, ok := r.definitions[jobType]
    return def, ok
}

// Types returns all registered job types.
func (r *Registry) Types() []JobType {
    r.mu.RLock()
    defer r.mu.RUnlock()

    types := make([]JobType, 0, len(r.definitions))
    for t := range r.definitions {
        types = append(types, t)
    }
    return types
}

// DefaultRegistry is the global default job registry.
var DefaultRegistry = NewRegistry()
```

### Errors (`errors.go`)

```go
package jobdef

import "errors"

// RetryableError wraps an error to indicate it should be retried.
type RetryableError struct {
    Err error
}

func (e *RetryableError) Error() string {
    return e.Err.Error()
}

func (e *RetryableError) Unwrap() error {
    return e.Err
}

// NewRetryableError creates a new retryable error.
func NewRetryableError(err error) *RetryableError {
    return &RetryableError{Err: err}
}

// IsRetryable checks if an error should trigger a retry.
func IsRetryable(err error) bool {
    var retryable *RetryableError
    return errors.As(err, &retryable)
}
```

## Package 2: Database Service (`jobsvc`)

### Service Interface

The job service handles all database operations for jobs.

```go
package jobsvc

import (
    "context"
    "encoding/json"
    "time"

    "github.com/fclairamb/solidping/back/internal/db/models"
)

// JobStatus represents the status of a job
type JobStatus string

const (
    JobStatusPending JobStatus = "pending"
    JobStatusRunning JobStatus = "running"
    JobStatusSuccess JobStatus = "success"
    JobStatusRetried JobStatus = "retried" // This job was retried (a new job was created)
    JobStatusFailed  JobStatus = "failed"
)

// Service provides job queue operations
type Service interface {
    // CreateJob creates a new job and notifies waiting runners
    CreateJob(ctx context.Context, orgUID, jobType string, config json.RawMessage) (*models.Job, error)

    // GetJob retrieves a job by UID
    GetJob(ctx context.Context, uid string) (*models.Job, error)

    // ListJobs lists jobs with optional filtering
    ListJobs(ctx context.Context, orgUID string, opts ListJobsOptions) ([]*models.Job, error)

    // CancelJob cancels a pending job (soft delete)
    CancelJob(ctx context.Context, uid string) error

    // GetJobWait waits for and claims the next available job
    // Uses PostgreSQL LISTEN/NOTIFY for efficient waiting
    GetJobWait(ctx context.Context) (*models.Job, error)

    // UpdateJobStatus updates job status to running/success/failed
    UpdateJobStatus(ctx context.Context, job *models.Job, status JobStatus, output json.RawMessage) error

    // RetryJob creates a new job from a failed job
    // Copies config, increments retry_count, sets previous_job_uid
    RetryJob(ctx context.Context, job *models.Job) (*models.Job, error)
}

// ListJobsOptions contains filtering options for listing jobs
type ListJobsOptions struct {
    Type   string
    Status string
    Limit  int
    Offset int
}
```

### Key Implementation Details

1. **Job Claiming** (`GetJobWait`):
   - Uses `SELECT ... FOR UPDATE SKIP LOCKED` for atomic claiming
   - Orders by: `scheduled_at ASC` (oldest first)
   - Filters: `scheduled_at <= NOW()`, `status = 'pending'`, `deleted_at IS NULL`
   - Sets `status = 'running'` when claimed
   - Uses PostgreSQL `LISTEN/NOTIFY` for efficient waiting (falls back to polling for SQLite)

2. **Duplicate Prevention**:
   - Before creating a job, check for existing job with same `type + config` that is pending
   - Return existing job if found (idempotent creation)

3. **Retry Logic** (`RetryJob`):
   - Mark original job as `retried`
   - Create new job with:
     - Same `organization_uid`, `type`, `config`
     - `retry_count = original.retry_count + 1`
     - `previous_job_uid = original.uid`
     - `scheduled_at = NOW()` (or exponential backoff)
     - `status = 'pending'`
   - Only retry if `retry_count < 3`

## Package 3: Worker Runner (`jobrunner`)

### Runner Structure

```go
package jobrunner

import (
    "context"
    "log/slog"

    "github.com/uptrace/bun"

    "github.com/fclairamb/solidping/back/internal/app/services"
    "github.com/fclairamb/solidping/back/internal/config"
    "github.com/fclairamb/solidping/back/internal/jobs/jobdef"
    "github.com/fclairamb/solidping/back/internal/jobs/jobsvc"
)

// Runner executes jobs from the queue
type Runner struct {
    id       int
    db       *bun.DB
    config   *config.Config
    services *services.ServicesList
    jobSvc   jobsvc.Service
    registry *jobdef.Registry
    logger   *slog.Logger
}

// NewRunner creates a new job runner
func NewRunner(
    id int,
    db *bun.DB,
    config *config.Config,
    services *services.ServicesList,
    jobSvc jobsvc.Service,
    registry *jobdef.Registry,
) *Runner {
    return &Runner{
        id:       id,
        db:       db,
        config:   config,
        services: services,
        jobSvc:   jobSvc,
        registry: registry,
        logger:   slog.Default().With("component", "job_runner", "runner_id", id),
    }
}

// Run starts the runner loop (blocking)
func (r *Runner) Run(ctx context.Context) error {
    r.logger.Info("Starting job runner")

    for {
        select {
        case <-ctx.Done():
            r.logger.Info("Job runner stopping")
            return ctx.Err()
        default:
            if err := r.processNext(ctx); err != nil {
                r.logger.Error("Error processing job", "error", err)
            }
        }
    }
}

func (r *Runner) processNext(ctx context.Context) error {
    // 1. Wait for and claim next job (sets status to 'running')
    job, err := r.jobSvc.GetJobWait(ctx)
    if err != nil {
        return err
    }

    // 2. Get job definition from registry
    def, ok := r.registry.Get(jobdef.JobType(job.Type))
    if !ok {
        r.jobSvc.UpdateJobStatus(ctx, job, jobsvc.JobStatusFailed,
            json.RawMessage(`{"error": "unknown job type"}`))
        return nil
    }

    // 3. Create job instance
    jobRun, err := def.CreateJobRun(job.Config)
    if err != nil {
        r.jobSvc.UpdateJobStatus(ctx, job, jobsvc.JobStatusFailed,
            json.RawMessage(fmt.Sprintf(`{"error": %q}`, err.Error())))
        return nil
    }

    // 4. Setup job context with logger
    jctx := r.createJobContext(job)

    // 5. Execute with panic recovery
    jobErr := r.executeWithRecovery(ctx, jobRun, jctx)

    // 6. Handle result (success, retry, or fail)
    return r.handleResult(ctx, job, jobErr)
}

func (r *Runner) executeWithRecovery(ctx context.Context, job jobdef.JobRun, jctx *jobdef.JobContext) (err error) {
    defer func() {
        if rec := recover(); rec != nil {
            err = fmt.Errorf("panic: %v", rec)
        }
    }()
    return job.Run(ctx, jctx)
}

func (r *Runner) handleResult(ctx context.Context, job *models.Job, err error) error {
    if err == nil {
        // Success: mark job as successful
        return r.jobSvc.UpdateJobStatus(ctx, job, jobsvc.JobStatusSuccess, nil)
    }

    if jobdef.IsRetryable(err) && job.RetryCount < 2 {
        // Retryable error and retries remaining: create retry job
        _, retryErr := r.jobSvc.RetryJob(ctx, job)
        if retryErr != nil {
            r.logger.Error("Failed to create retry job", "job_uid", job.UID, "error", retryErr)
        }
        // Mark original as retried
        return r.jobSvc.UpdateJobStatus(ctx, job, jobsvc.JobStatusRetried,
            json.RawMessage(fmt.Sprintf(`{"error": %q, "retried": true}`, err.Error())))
    }

    // Non-retryable error or max retries reached: mark as failed
    return r.jobSvc.UpdateJobStatus(ctx, job, jobsvc.JobStatusFailed,
        json.RawMessage(fmt.Sprintf(`{"error": %q}`, err.Error())))
}
```

type SleepJobDefinition struct{}

func (d *SleepJobDefinition) Type() jobdef.JobType {
    return jobdef.JobTypeSleep

func (d *SleepJobDefinition) CreateJobRun(config json.RawMessage) (jobdef.JobRun, error) {
    var cfg SleepJobConfig
    if err := json.Unmarshal(config, &cfg); err != nil {
        return nil, err
    }
    return &SleepJobRun{config: cfg}, nil
}

type SleepJobRun struct {
    config SleepJobConfig
}

func (r *SleepJobRun) Run(ctx context.Context, jctx *jobdef.JobContext) error {
    jctx.Logger.Info("Starting sleep job", "duration_seconds", r.config.Seconds)

    select {
    case <-ctx.Done():
        return ctx.Err()
    case <-time.After(time.Duration(r.config.Seconds) * time.Second):
        jctx.Logger.Info("Sleep completed")
        return nil
    }
}

### Email Job

```go
package jobtypes

import (
    "context"
    "encoding/json"
    "fmt"
    "net/smtp"

    "github.com/fclairamb/solidping/back/internal/jobs/jobdef"
)

func init() {
    jobdef.DefaultRegistry.Register(&EmailJobDefinition{})
}

type EmailJobConfig struct {
    To      string `json:"to"`
    Subject string `json:"subject"`
    Body    string `json:"body"`
}

type EmailJobDefinition struct{}

func (d *EmailJobDefinition) Type() jobdef.JobType {
    return jobdef.JobTypeEmail
}

func (d *EmailJobDefinition) CreateJobRun(config json.RawMessage) (jobdef.JobRun, error) {
    var cfg EmailJobConfig
    if err := json.Unmarshal(config, &cfg); err != nil {
        return nil, err
    }
    return &EmailJobRun{config: cfg}, nil
}

type EmailJobRun struct {
    config EmailJobConfig
}

func (r *EmailJobRun) Run(ctx context.Context, jctx *jobdef.JobContext) error {
    jctx.Logger.Info("Sending email", "to", r.config.To, "subject", r.config.Subject)

    smtpCfg := jctx.AppConfig.SMTP
    if smtpCfg.Host == "" {
        return fmt.Errorf("SMTP not configured")
    }

    auth := smtp.PlainAuth("", smtpCfg.Username, smtpCfg.Password, smtpCfg.Host)

    msg := fmt.Sprintf("Subject: %s\r\n\r\n%s", r.config.Subject, r.config.Body)

    err := smtp.SendMail(
        fmt.Sprintf("%s:%d", smtpCfg.Host, smtpCfg.Port),
        auth,
        smtpCfg.From,
        []string{r.config.To},
        []byte(msg),
    )

    if err != nil {
        // Network errors are retryable
        return jobdef.NewRetryableError(err)
    }

    jctx.Logger.Info("Email sent successfully")
    return nil
}
```

func (r *CheckRunJobRun) Run(ctx context.Context, jctx *jobdef.JobContext) error {
    jctx.Logger.Info("Running check", "check_uid", r.config.CheckUID)

    // Get check from database
    check, err := jctx.Services.DB.GetCheck(ctx, r.config.CheckUID)
    if err != nil {
        return err
    }

    // Execute check based on type
    // This would call the appropriate checker service
    result, err := jctx.Services.Checker.Execute(ctx, check)

        return err
    }

    jctx.Logger.Info("Check completed", "status", result.Status, "duration_ms", result.DurationMsAvg)
    return nil

## Database Schema

### Jobs Table

```sql
CREATE TABLE jobs (
    uid UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    organization_uid UUID NOT NULL REFERENCES organizations(uid),
    type TEXT NOT NULL CHECK (type ~ '^[a-z0-9-]{3,}$'),
    config JSONB,
    retry_count INT NOT NULL DEFAULT 0,
    scheduled_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    status TEXT NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'running', 'success', 'retried', 'failed')),
    output JSONB,
    previous_job_uid UUID REFERENCES jobs(uid),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    deleted_at TIMESTAMPTZ
);

-- Index for job queue claiming
CREATE INDEX idx_jobs_queue ON jobs(scheduled_at, status)
    WHERE deleted_at IS NULL AND status = 'pending';

-- Index for organization queries
CREATE INDEX idx_jobs_organization ON jobs(organization_uid, created_at DESC)
    WHERE deleted_at IS NULL;

-- Index for retry chain lookups
CREATE INDEX idx_jobs_previous ON jobs(previous_job_uid)
    WHERE previous_job_uid IS NOT NULL;

-- PostgreSQL notification trigger for new jobs
CREATE OR REPLACE FUNCTION notify_new_job() RETURNS TRIGGER AS $$
BEGIN
    PERFORM pg_notify('new_job', NEW.uid::TEXT);
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_notify_new_job
    AFTER INSERT ON jobs
    FOR EACH ROW
    WHEN (NEW.status = 'pending')
    EXECUTE FUNCTION notify_new_job();
```

## Key Design Decisions

1. **Single table design**: Simple schema without separate executions or logs tables
2. **Standard logging with correlation**: Logs written to stdout/stderr with `jobUid` attribute for filtering
3. **Retry as new job**: Each retry creates a new job row linked via `previous_job_uid`
4. **No separate executions**: Job status transitions directly (pending → running → success/failed/retried)
5. **PostgreSQL LISTEN/NOTIFY**: Efficient job notification for queue workers
6. **Organization-scoped**: All jobs belong to an organization
7. **Soft deletes**: Jobs can be cancelled via `deleted_at`
8. **Status `retried`**: Marks jobs that triggered a retry (vs final states success/failed)
